# 🚀 Quick Start Guide — PuLID Training

## 📋 Prerequisites

### 1. Environment Setup

```bash
# PyTorch with CUDA 12.4
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124

# Core dependencies
pip install accelerate diffusers transformers
pip install open-clip-torch insightface safetensors
pip install opencv-python pillow tqdm

# Verify CUDA
python -c "import torch; print(f'CUDA: {torch.cuda.is_available()} | {torch.cuda.get_device_name(0)}')"
```

> ⚠️ Do **NOT** install `eva_clip` from GitHub — use `open-clip-torch` only.

### 2. Folder Structure


### 3. Dataset Requirements

| | Minimum | Recommended | Ideal |
|---|---|---|---|
| Identities | 50 | 200 | 500+ |
| Photos/identity | 5 | 10 | 10-20 |
| Total images | 250 | 2 000 | 5 000-10 000 |

**Quality criteria:**
- HD photos (>512px)
- Face clearly visible and well-lit
- Variety: angles, expressions, lighting conditions
- No masks / sunglasses / heavy occlusions

## 🎯 Phase 1 — IDFormer Training

### Goal
Learn to create **rich identity tokens** that capture the essence of a face from InsightFace (512-d) + EVA-CLIP (768-d) embeddings.

### Simple Command

```bash
python train_pulid.py \
    --dataset ./dataset/filtered \
    --output ./output \
    --phase 1
```

Auto-configured defaults: 25 epochs, batch=4, lr=1e-4, image_size=512

### Full Command

```bash
python train_pulid.py \
    --dataset ./dataset/filtered \
    --output ./output \
    --comfyui_path /path/to/ComfyUI \
    --phase 1 \
    --dim 4096 \
    --epochs 30 \
    --batch_size 8 \
    --lr 1e-4 \
    --save_every 500
```

### Monitoring Phase 1

```
Epoch 1:   loss_identity=0.280  loss_reg=0.008  ← Starting
Epoch 10:  loss_identity=0.085  loss_reg=0.005  ← Progressing
Epoch 20:  loss_identity=0.042  loss_reg=0.004  ← ✅ Good
Epoch 25:  loss_identity=0.028  loss_reg=0.003  ← ✅ Excellent
```

**Success criteria:**
- ✅ `loss_identity < 0.05` → Good, can move to Phase 2
- ✅ `loss_identity < 0.03` → Excellent, optimal results
- ⚠️ `loss_identity > 0.10` → Not converged yet, keep training
- ❌ `loss_identity stalls > 0.15` → Dataset or config issue

### Estimated Duration (Phase 1, 25 epochs)

|---|---|---|
| RTX 3060 12GB | 2 | 4–6 hours |
| RTX 3090 24GB | 4 | 2–3 hours |
| RTX 4090 24GB | 8 | 1–2 hours |
| A100 40GB | 16 | ~1 hour |

### Troubleshooting Phase 1

**CUDA Out of Memory:**
```bash
python train_pulid.py --phase 1 --batch_size 2
```

**Loss not decreasing:**
```bash
# Check dataset size
python -c "
import json
with open('./dataset/filtered/metadata.json') as f:
    meta = json.load(f)
print(f'{meta[\"total_kept\"]} images loaded')
"
# If <200 images: collect more photos
```

**Loss decreases then increases (overfitting):**
The script automatically saves the best checkpoint — use `phase1_best.safetensors`, not `latest`.

## 🎯 Phase 2 — PerceiverCA Training

### Goal
Learn to **inject identity** into Flux.2 image tokens via cross-attention corrections on both double_blocks and single_blocks.

### Prerequisites

1. Phase 1 checkpoint: `output/pulid_klein_phase1_best.safetensors`
2. Flux.2 Klein model:
   ```bash
   # Klein 9B (recommended)
   huggingface-cli download black-forest-labs/FLUX.2-Klein-9B \
       --local-dir ./models/flux2-klein-9b

   # Or use a single .safetensors file
   # flux-2-klein-9b-fp8.safetensors
   ```

### Simple Command

```bash
python train_pulid.py \
    --dataset ./dataset/filtered \
    --output ./output_phase2 \
    --phase 2 \
    --resume ./output/pulid_klein_phase1_best.safetensors \
    --flux_model_path ./models/flux2-klein-9b
```

Auto-configured defaults: 15 epochs, batch=2, lr=5e-5

### Full Command

```bash
python train_pulid.py \
    --dataset ./dataset/filtered \
    --output ./output_phase2 \
    --phase 2 \
    --resume ./output/pulid_klein_phase1_best.safetensors \
    --flux_model_path ./models/flux-2-klein-9b-fp8.safetensors \
    --comfyui_path /path/to/ComfyUI \
    --dim 4096 \
    --epochs 20 \
    --batch_size 1 \
    --lr 3e-5
```

### Monitoring Phase 2

```
Epoch 1:   loss_identity=0.045  loss_perceiver=0.018  ← Starting
Epoch 5:   loss_identity=0.038  loss_perceiver=0.008  ← Descending
Epoch 10:  loss_identity=0.032  loss_perceiver=0.005  ← ✅ Good
Epoch 15:  loss_identity=0.028  loss_perceiver=0.004  ← ✅ Excellent
```

**Success criteria:**
- ✅ `loss_identity < 0.03` → Excellent
- ✅ `loss_perceiver < 0.01` → PerceiverCA well trained
- ⚠️ `loss_identity increasing` → Check Phase 1 quality or lower lr
- ⚠️ `loss_perceiver > 0.02` → Not converged, add epochs

### Estimated Duration (Phase 2, 15 epochs)

| GPU | Batch Size | Duration |
|---|---|---|
| RTX 3060 12GB | 1 | 16–20 hours |
| RTX 3090 24GB | 2 | 8–12 hours |
| RTX 4090 24GB | 3 | 6–8 hours |
| A100 40GB | 4 | 4–6 hours |

> ⚠️ Phase 2 is significantly slower than Phase 1 (Flux.2 loaded in VRAM even frozen).

### Troubleshooting Phase 2

**CUDA Out of Memory:**
```bash
# Reduce batch size
--batch_size 1

# Or use Klein 4B instead of 9B
--dim 3072 --flux_model_path ./models/flux-2-klein-4b-fp8.safetensors
```

**loss_identity increasing:**
```bash
# Verify Phase 1 checkpoint
python -c "
from safetensors.torch import load_file
state = load_file('./output/pulid_klein_phase1_best.safetensors')
print(f'Phase 1 checkpoint loaded: {len(state)} keys')
print(f'id_former dim: {state[\"id_former.latents\"].shape}')
"
# If Phase 1 was poor: redo Phase 1 first
```

**loss_perceiver not decreasing:**
```bash
# Lower learning rate
--lr 2e-5
# Or add epochs
--epochs 25
```

## 🧪 Testing in ComfyUI

### 1. Copy Checkpoint

```bash
cp output_phase2/pulid_klein_phase2_best.safetensors \
   ComfyUI/models/pulid/pulid_klein_custom.safetensors
```

### 2. Recommended Parameters (after training native weights)

```
weight:     1.0    ← Higher than with Flux.1 weights
start_at:   0.0
end_at:     0.5    ← Critical: prevents darkening
face_index: 0
```

> With Flux.1 weights (before native training), use `weight: 0.6, end_at: 0.5`.  
> With your own trained weights, try `weight: 1.0–1.2, end_at: 0.5–0.7`.

### 3. Progressive Tests

**Test 1 — Pure Identity**
```
Image Ref : Your selfie
Prompt    : "professional headshot, neutral background, studio lighting"
Weight    : 1.0 / end_at: 0.5
Expected  : Face IDENTICAL to reference
```

**Test 2 — Style Change**
```
Image Ref : Your selfie
Prompt    : "portrait at sunset by the river, golden hour"
Weight    : 0.9 / end_at: 0.5
Expected  : SAME face, new context
```

**Test 3 — Radical Change**
```
Image Ref : Your selfie
Prompt    : "cyberpunk character, neon city, futuristic"
Weight    : 1.2 / end_at: 0.6
Expected  : Recognizable face, different style
```

### 4. Parameter Reference

| Parameter | Range | Effect |
|---|---|---|
| `weight` | 0.5–0.7 | Subtle identity, more creative freedom |
| | 0.8–1.0 | ✅ Balanced (recommended) |
| | 1.0–1.5 | Strong identity, less variation |
| | >1.5 | ⚠️ Risk of artifacts |
| `end_at` | 0.4 | Subtle influence, max natural lighting |
| | **0.5** | **✅ Optimal (prevents darkening)** |
| | 0.6 | Stronger influence, slight darkening risk |
| | >0.7 | ⚠️ Darkening artifacts |

## ❓ FAQ

**Q: Total training time?**  
A: Phase 1 (~3h) + Phase 2 (~10h) = ~13h on RTX 3090.

**Q: Can I skip Phase 1?**  
A: No. Phase 1 is mandatory — Phase 2 without Phase 1 will fail.

**Q: Minimum GPU?**  
A: 12GB VRAM (RTX 3060) with `batch_size=1`. Recommended: 24GB (RTX 3090).

**Q: Klein 4B vs Klein 9B?**  
A: Klein 9B = better quality. Klein 4B = faster, less VRAM required.

**Q: How many photos per person?**  
A: Minimum 5, ideal 10–20 with variety (angles, expressions, lighting).

**Q: Can I use low-quality photos?**  
A: Yes but results will be degraded. Prefer >512px, well-lit images.

**Q: Early stopping at 5 epochs — is that enough?**  
A: Yes. The best checkpoint is saved automatically. `patience=5` prevents overfitting.
